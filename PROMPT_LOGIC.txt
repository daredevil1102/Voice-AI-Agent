# Voice AI Clinic Assistant - Dialog & State Logic Analysis

This document details the underlying dialog logic used to drive the bilingual receptionist agent for Aarohan Clinics.

---

## 1. Caller Identification Flow
At call initiation, the agent automatically executes the `identify_caller` tool:
- **Dropped Call Recovery:** If an active session is found in the database within the last 10 minutes containing booking details, it returns `dropped_call_recovery`. The agent acknowledges the cutoff and asks if they want to resume the booking.
- **New Patient:** If no profile is matched to the phone number, it returns `new_patient`. The agent welcomes them and asks for their full name to get started.
- **Family Line Disambiguation:** If multiple patients share the calling number (e.g., kids/parents sharing a number), it returns `family_line_shared`. The agent prompts the caller for their full name to access the correct record.
- **Returning Patient:** If exactly one patient matches the number, it returns `returning_patient`. The agent greets them by name and asks how they can help.

---

## 2. Booking & Name Capture Enforcement
- **Security Check:** To prevent anonymous bookings, the agent's prompt logic instructs it to always capture or verify the caller's first and last name before calling the `book_appointment` endpoint, even if the phone number was already matched to a profile.
- **Conflict Management:** If a slot is booked concurrently by someone else, the tool returns `success: false` with the conflict reason. The agent is trained to handle this fallback gracefully and run a live availability check for the next slot.

---

## 3. Bilingual English/Hindi Mid-Call Code-Switching
- **Mirroring Behavior:** The prompt instructs the agent to dynamically mirror the user's language choice.
- **Natural Mid-Sentence Switching (Hinglish):** It leverage native multilingual tokens on the LLM (GPT-4o) directly. For example, if a user speaks Hindi, the agent responds in clean, conversational Hinglish (e.g., "Kya aapko subah ka slot chahiye ya afternoon ka?"). This avoids the latency of double-transcription translation middlewares.

---

## 4. Late Fee Announcement Logic (Rescheduling & Cancellation)
- **Fee Warnings on Request:** The agent is explicitly instructed NOT to announce rescheduling/cancellation fees by default.
- **Conditional Trigger:** When a reschedule or cancellation tool is called, the backend calculates the difference between current UTC time and the scheduled appointment. If it is less than 24 hours, the tool returns `fee_applies: true` and `fee_amount: "$25"`. The agent only states the fee and requests confirmation when these values are returned.

---

## 5. Redundancy Prevention (Contextual Memory)
- **Heuristic Memory:** The agent maps multiple parameters from a single user turn (e.g., "Dr. Ramesh tomorrow morning") directly to the availability check tool, skipping intermediate confirmation questions.
- **Re-ask Blocker:** The system prompt blocks the agent from re-asking questions whose answers already exist in the message history.
